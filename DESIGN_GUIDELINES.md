| Neutral-300 | #d9d9d9 | 中灰色，用于分割线 |

| 圆角大小 | 用途 |

| 16px | 特殊卡片、模态框 |

### 1.5 阴影规范

| 阴影类型 | CSS | 用途 |

## 2. 组件设计规范

### 2.1 按钮组件

| 类型 | 样式 | 用途 |
| --- | --- | --- |
| 主要按钮 | 主色背景，白色文字 | 主要操作，如提交、保存 |
| 次要按钮 | 白色背景，主色边框和文字 | 次要操作，如取消、返回 |
| 危险按钮 | 红色背景，白色文字 | 危险操作，如删除 |
| 文本按钮 | 无背景，主色文字 | 轻量级操作，如链接 |

### 2.2 表单组件

| 组件 | 样式 | 用途 |
| --- | --- | --- |
| 输入框 | 浅灰色边框，聚焦时主色边框 | 文本输入 |
| 下拉选择 | 与输入框一致，右侧带箭头图标 | 选择选项 |
| 日期选择 | 与输入框一致，右侧带日历图标 | 选择日期 |
| 复选框 | 方形，选中时主色背景和白色对勾 | 多选 |
| 单选框 | 圆形，选中时主色背景和白色圆点 | 单选 |

### 2.3 卡片组件

| 类型 | 样式 | 用途 |
| --- | --- | --- |
| 标准卡片 | 白色背景，浅灰色边框，小阴影 | 展示信息 |
| 强调卡片 | 白色背景，主色边框，中阴影 | 重要信息 |
| 统计卡片 | 白色背景，顶部带图标，小阴影 | 数据统计 |

### 2.4 表格组件

| 元素 | 样式 | 用途 |
| --- | --- | --- |
| 表头 | 浅灰色背景，加粗文字 | 表格标题 |
| 行 | 白色背景，hover 时浅灰色背景 | 数据行 |
| 分割线 | 浅灰色，1px 高度 | 分隔行 |
| 分页 | 与按钮样式一致 | 翻页控制 |

## 3. 页面设计规范

### 3.1 布局结构

- **顶部导航栏**：固定在顶部，包含系统名称、导航菜单和用户信息
- **侧边栏**：在桌面端显示，包含主要功能导航
- **主内容区**：自适应宽度，包含页面内容
- **底部信息**：包含版权信息和相关链接

### 3.2 响应式设计

| 屏幕尺寸 | 布局调整 |
| --- | --- |
| 移动端 (< 768px) | 隐藏侧边栏，使用底部导航或汉堡菜单 |
| 平板端 (768px - 1024px) | 侧边栏可折叠，内容区自适应 |
| 桌面端 (> 1024px) | 完整布局，侧边栏固定显示 |

### 3.3 页面类型

| 页面类型 | 设计要点 |
| --- | --- |
| 列表页 | 表格展示，支持筛选、排序和分页 |
| 表单页 | 清晰的表单布局，合理的字段分组 |
| 详情页 | 卡片式布局，信息层次分明 |
| 统计页 | 图表展示，数据可视化清晰 |

## 4. TDD 开发模式的设计规范

### 4.1 组件测试策略

- **每个组件一个测试文件**：与组件文件放在同一目录下
- **测试覆盖范围**：
  - 组件渲染正常
  - 数据加载与处理
  - 用户交互响应
  - 边界情况处理
  - 错误状态处理

### 4.2 设计与测试的关系

1. **先设计，后测试**：基于设计规范创建组件
2. **测试驱动设计**：通过测试用例验证设计是否符合预期
3. **设计反馈**：测试结果可用于优化设计

### 4.3 组件目录结构

```

│   ├── index.vue         # 组件实现
│   └── StallManagement.spec.js  # 测试文件
```

## 5. 交互设计规范

### 5.1 微交互

- **按钮悬停**：轻微的背景色变化和阴影增强
- **表单输入**：聚焦时边框颜色变化，错误时红色边框和提示
- **加载状态**：骨架屏或加载动画
- **成功反馈**：短暂的成功提示或动画

### 5.2 导航设计

- **主导航**：清晰的菜单项，当前页面高亮显示
- **面包屑**：显示当前页面路径，便于导航
- **返回按钮**：在详情页和表单页提供返回功能

### 5.3 反馈机制

- **操作成功**：绿色提示框，自动消失
- **操作失败**：红色提示框，显示错误信息
- **确认对话框**：危险操作前的确认提示
- **加载提示**：操作进行中的加载动画

## 6. 可访问性规范

### 6.1 颜色对比度

- 文本与背景的对比度至少为 4.5:1
- 大文本（18px 以上）的对比度至少为 3:1

### 6.2 键盘导航

- 所有交互元素可通过键盘访问
- 清晰的焦点状态
- 合理的 Tab 键顺序

### 6.3 屏幕阅读器

- 所有图片有 alt 文本
- 表单元素有标签
- 动态内容有适当的 ARIA 属性

## 7. 实现指南

### 7.1 技术栈

- **前端框架**：Vue 3
- **UI 库**：Ant Design Vue
- **样式**：CSS/SCSS
- **测试**：Vitest + Vue Test Utils

### 7.2 开发流程

1. **设计阶段**：基于设计规范创建组件设计
2. **测试阶段**：为组件编写测试用例
3. **实现阶段**：根据设计和测试实现组件
4. **验证阶段**：运行测试确保组件符合预期
5. **优化阶段**：根据测试结果和用户反馈优化设计

### 7.3 性能优化

- 组件懒加载
- 图片优化
- 减少不必要的重渲染
- 合理使用缓存

## 8. 交付标准

### 8.1 代码质量

- 遵循 ESLint 规范
- 代码注释清晰
- 组件结构合理
- 测试覆盖率达到 80% 以上

### 8.2 设计一致性

- 符合设计规范
- 视觉效果统一
- 交互体验一致
- 响应式适配良好

### 8.3 文档完善

- 组件使用文档
- 测试用例说明
- 设计决策记录
- 变更日志

## 9. 维护与更新

### 9.1 设计系统维护

- 定期更新设计规范
- 组件库版本管理
- 设计资源管理

### 9.2 测试用例维护

- 组件变更时更新测试用例
- 新增功能时添加测试用例
- 定期运行测试确保系统稳定性

此设计规范旨在为蔬菜接单系统提供统一的设计标准，确保系统的视觉一致性和用户体验。同时，通过 TDD 开发模式，保证代码质量和功能可靠性。
